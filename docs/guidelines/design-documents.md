## Design document references

Every design document in `docs/plans/` must include a **References** section **at the end of the document**, listing the source files that informed the design. This ensures traceability — future brainstorms and agents can find the original context that shaped each decision.

What to include:
- File paths within the repo (e.g., `tmp/rwa-v1/...`, `tmp/rwa-v1-dive/...`)
- A one-line summary of what each reference contributed
- A **key takeaway** synthesizing how the references shaped the design

Placement: always the **last section** of the design document. This convention makes design documents self-contained — readers don't need to guess which v1 branches, dive analyses, or prior plans were consulted.
